🏗️ HTML：网页的骨架

网页，就像一座华丽的宫殿。而HTML（Hypertext Markup Language），就是这座宫殿的地基和骨架！没有坚固的地基和合理的骨架，再漂亮的装饰也撑不起来！

HTML文档基本结构：

<!DOCTYPE html>  <!-- 告诉浏览器这是HTML5文档，这是规矩！ -->
<html lang="zh-CN"> <!-- 整个网页的根元素，指定语言 -->
<head>          <!-- 网页的"头部"，放一些设置和元信息 -->
    <meta charset="UTF-8"> <!-- 指定字符编码，避免乱码！ -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>本小姐的网页标题</title>
</head>
<body>          <!-- 网页的"身体"，所有用户能看到的内容 -->
    <!-- 你的网页内容 -->
</body>
</html>

🚀 Node.js 初体验

Node.js 是一个开源的JavaScript运行时环境。你可以把它想象成一个"容器"，把你写的JavaScript代码放进去，它就能在你的电脑上或者服务器上运行起来！最棒的是，你不用学新的语言，还是用JavaScript！

第一个 Node.js 服务器：

// 导入 Node.js 内置的 http 模块
const http = require('http');

const hostname = '127.0.0.1';
const port = 3000;

// 创建 HTTP 服务器
const server = http.createServer((req, res) => {
  // 设置响应头
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain; charset=utf-8');
  
  // 发送响应内容
  res.end('哼，这是本小姐的第一个Node.js服务器响应！\n');
});

// 启动服务器监听指定端口
server.listen(port, hostname, () => {
  console.log(`服务器运行在 http://${hostname}:${port}/`);
  console.log('本小姐的服务器已经准备好接客了！');
});

🎯 Express.js 闪亮登场

虽然Node.js内置的http模块能搞定基本需求，但要是想更轻松地处理复杂的路由、中间件、静态文件等等，就需要更强大的工具了！Express.js就是其中的佼佼者！

Express 基础应用：

const express = require('express');
const app = express();
const port = 3000;

// 中间件：解析JSON请求体
app.use(express.json());

// 根路由
app.get('/', (req, res) => {
  res.send('哼！这是本小姐用 Express 搭建的服务器！');
});

// 获取用户信息
app.get('/users/:userId', (req, res) => {
  const userId = req.params.userId;
  res.json({
    message: `你访问的是用户 ID 为 ${userId} 的信息！`,
    id: userId,
    name: "本小姐的用户",
    level: "初心者",
    status: "活跃"
  });
});

// 创建新用户
app.post('/users', (req, res) => {
  const { name, email } = req.body;
  res.json({
    message: '新用户创建成功！',
    user: { id: Date.now(), name, email },
    time: new Date().toISOString()
  });
});

app.listen(port, () => {
  console.log(`Express 应用运行在 http://localhost:${port}/`);
  console.log('本小姐的Express帝国正式开张！');
});

💎 第五章：数据宝库守护——你的小秘密，本小姐帮你藏好！

哼，你那些重要的、不想让别人随便看到的"小秘密"（数据），总不能随便乱放吧？也不能等服务器一关机就全丢了！所以，我们需要一个安全、有条理的地方来"保管"它们！这就是数据库（Database）的作用！

📊 关系型数据库 (SQL)

最传统、最常见的数据库类型。把数据组织成表格的形式。每个表格有固定的列，每一行代表一条记录。

典型代表：MySQL, PostgreSQL, SQLite

📄 非关系型数据库 (NoSQL)

更灵活的存储方式，比如文档型数据库以JSON格式存储数据，结构非常灵活。

典型代表：MongoDB, Redis, Firebase

数据库基础操作（CRUD）：

CREATE (创建)

往数据库里添加新数据

READ (读取)

从数据库里查找和获取数据

UPDATE (更新)

修改已有的数据

DELETE (删除)

从数据库里移除数据

SQL基础操作概念：

-- 查询数据 (SELECT/READ)
SELECT name, age, email FROM users WHERE age > 18;

-- 插入数据 (INSERT/CREATE)  
INSERT INTO users (name, age, email) 
VALUES ('本小姐', 18, 'ojousama@example.com');

-- 更新数据 (UPDATE)
UPDATE users 
SET age = 19, status = '活跃' 
WHERE name = '本小姐';

-- 删除数据 (DELETE)
DELETE FROM users WHERE age < 13;

-- 创建表结构
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL,
    age INT,
    email VARCHAR(100) UNIQUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

🎭 RESTful API设计理念

REST (Representational State Transfer) 是一种设计API的风格和规范。它让API的使用更加直观和标准化。

GET /api/users

获取所有用户列表

POST /api/users

创建新用户

PUT /api/users/123

更新ID为123的用户

DELETE /api/users/123

删除ID为123的用户

前端调用API示例（Fetch API）：

// 获取用户数据
async function getUsers() {
  try {
    const response = await fetch('/api/users', {
      method: 'GET',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': 'Bearer your-token-here'
      }
    });

    if (!response.ok) {
      throw new Error(`哼！请求失败了，状态码：${response.status}`);
    }

    const data = await response.json();
    console.log('成功从后端获取到用户数据：', data);
    
    // 更新页面显示
    displayUsers(data);
    
  } catch (error) {
    console.error('向后端"撒娇"要数据失败：', error);
    showErrorMessage('获取数据失败，请稍后再试！');
  }
}

// 创建新用户
async function createUser(userData) {
  try {
    const response = await fetch('/api/users', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify(userData)
    });

    const result = await response.json();
    console.log('新用户创建成功：', result);
    
  } catch (error) {
    console.error('创建用户失败：', error);
  }
}

🌍 域名与DNS

域名就是你网站的"地址牌"，比如 www.example.com。DNS（Domain Name System）就是把这个好记的地址翻译成服务器实际IP地址的"翻译官"。
